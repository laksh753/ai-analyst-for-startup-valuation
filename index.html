Problem Statement

The startup ecosystem faces a 55.6% failure rate globally, with only 2.1% achieving IPOs. Investors and accelerators lack quantitative early-warning systems to predict outcome trajectories. This project builds a multi-class classification model to predict whether a startup will be Acquired, Fail, or IPO based on funding stage, team composition, market metrics, and revenue signals — enabling data-driven investment decisions.

Literature Review

Existing work (Zheng et al., 2020; Arroyo et al., 2019) focuses on binary success/failure prediction using limited features (<1K samples). None address multi-class outcomes (Acquisition vs IPO vs Failure) at scale with sector-agnostic features. CB Insights (2023) relies on qualitative scoring — no ML approach exists for 100K+ diverse startups.

Research Gap / Innovation

This is the first study to: (1) apply ensemble learning (Random Forest) to 100K startup records, (2) predict three distinct outcomes rather than binary classification, and (3) identify that revenue_million is 47% more predictive than any other feature — a novel empirical finding.

Key Findings

  • • Revenue is the #1 predictor (importance 0.2899) — 47% stronger than 2nd feature
  • • Product traction (users) is 2nd most important (0.1968)
  • • Random Forest outperformed Logistic Regression by +8.6 percentage points
  • • Model correctly predicts 82% of Failures (high recall on majority class)
  • • IPO prediction is challenging due to rarity (2.1% of data) — only 19% recall

Limitations

  • • Class imbalance: IPO is only 2.1% of dataset — hard to learn rare patterns
  • • LabelEncoding imposes ordinal relationships on nominal categories
  • • Synthetic data may not capture all real-world complexity (e.g., founder network effects)
  • • No temporal validation — older vs newer startups not separated

Future Scope

  • • Apply SMOTE (Synthetic Minority Oversampling) for IPO class
  • • Try XGBoost / LightGBM for gradient boosting
  • • Feature engineering: revenue/employee ratio, burn rate/funding ratio
  • • Hyperparameter tuning via GridSearchCV
  • • Deploy as REST API with FastAPI for real-time predictions

AI Startup Profitability Predictor

Problem Statement

The global AI startup ecosystem is growing exponentially, yet over 70% of AI ventures fail to reach profitability within their first five years. Investors, accelerators, and founders lack a data-driven early-warning signal to gauge viability. This project addresses that gap by building a binary classification system that predicts startup profitability from structured funding metadata — before market outcomes are known.

Literature Review

Existing studies (CB Insights 2023, Crunchbase AI Report 2024) rely on qualitative expert scoring. Supervised ML approaches using Crunchbase features (Antretter et al., 2019) achieved ~62 % AUC on general startups but were never applied to the AI-domain subset with AI-specific features such as AI model type and funding stage.

Research Gap / Innovation

No prior work combines AI-domain-specific features (AI Model Type, Industry AI Application) with classical funding signals to predict profitability. Our dataset is curated exclusively for the AI sector (2020–2023), enabling sector-specific pattern detection beyond generic startup predictors.

Key Findings

  • • AI Model Type is the single most predictive feature (importance 0.182)
  • • Industry domain is the second strongest signal (0.167)
  • • Logistic Regression outperformed Random Forest on this small dataset
  • • High recall (0.88) for "Not Profitable" class — good at catching failures

Limitations

  • • Only 60 records — insufficient for robust generalisation
  • • Class imbalance: 63% Not Profitable vs 37% Profitable
  • • LabelEncoding imposes artificial ordinal relationships
  • • No temporal train/test split — data leakage risk

Future Scope
